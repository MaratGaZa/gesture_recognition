# План реализации

## Обзор
Данный план описывает конкретные шаги, необходимые для создания веб‑приложения, распознающего жесты руки и выводящего эмодзи, согласно документации в папке `docs/`.  
Для каждой задачи указаны **описание**, **Definition of Done (DoD)** и поле **статус**, которое можно обновлять по мере выполнения.

---

## Задачи

### 2. Реализация модуля **Camera** (`src/core/camera.js`)
**Описание**  
Должена быть возможность выбора камеры из списка доступных устройств и запуск камеры на переданном элементе `<video>`.  
При выборе камеры должна отображаться её имя в UI.

**Definition of Done**  
- Функция `initCamera(videoEl): Promise<void>` корректно выбирает
Экспортировать функцию `initCamera(videoEl): Promise<void>`, которая:
- Запрашивает доступ к веб‑камере.  
- Передаёт видеопоток в переданный элемент `<video>`.  
- Обрабатывает ошибки (например, отказ в доступе) и отклоняет промис с понятным сообщением.

**Definition of Done**  
- Функция возвращает выполненный промис, когда видео успешно воспроизводится.  
- Ошибки логируются, промис отклоняется объектом `Error`.  
- Написан юнит‑тест, проверяющий как успешный, так и ошибочный сценарий (мок `navigator.mediaDevices`).

**Статус**: TODO

### 3. Реализация модуля **Loop** (`src/core/loop.js`)
**Описание**  
Экспортировать `startLoop(callback)`, который запускает `callback` на каждом кадре с помощью `requestAnimationFrame`.  
Цикл **не содержит бизнес‑логики** – только управляет процессом.

**Definition of Done**  
- `startLoop` возвращает объект‑хэндл с методом `stop()`.  
- `callback` получает метку времени (`time`).  
- Внутри цикла нет манипуляций с DOM.  
- Юнит‑тест подтверждает, что `callback` вызывается многократно и может быть остановлен.

**Статус**: TODO

---

### 4. Реализация **HandTracker** (`src/vision/handTracker.js`)
**Описание**  
Обёртка вокруг MediaPipe Hand Landmarker:
- Инициализация модели (`init(): Promise<void>`).  
- Метод `detect(video, time): { landmarks }`, возвращающий массив из 21 координаты руки.

**Definition of Done**  
- Модель загружается без ошибок.  
- `detect` возвращает объект с `landmarks` либо `null`, если рука не найдена.  
- Ошибки отлавливаются и передаются наружу.  
- Интеграционный тест с заглушкой видеэлемента проверяет корректность получения landmarks.

**Статус**: TODO

### 5. Реализация **Gesture Engine** (`src/gestures/gestureEngine.js`)
**Описание**  
Чистая функция `detectGesture(landmarks): string | null`, которая:
- Делегирует проверку отдельным модулям (`thumbsUp.js`, `rock.js`).  
- Возвращает константу (`THUMBS_UP`, `ROCK`) или `null`, если известный жест не обнаружен.

**Definition of Done**  
- Нет побочных эффектов и работы с DOM.  
- Все файлы‑жесты импортированы и вызываются.  
- Полный набор юнит‑тестов покрывает каждый жест и случай отсутствия жеста.

**Статус**: TODO

### 7. Реализация **ReactionState** (`src/state/reactionState.js`)
**Описание**  
Класс `ReactionState` с методом:
- `update(gesture): string | null` – возвращает жест только если он прошёл проверку **cooldown** и не является дублем.  
Внутри реализованы:
- **debounce** (откат повторов).  
- **cooldown** (например, 500 мс).  
- Подавление повторных срабатываний.

**Definition of Done**  
- Константа `COOLDOWN_MS` конфигурируема.  
- Правильно фильтрует быстрые повторы.  
- Юнит‑тесты проверяют поведение во времени с мок‑таймерами.

**Статус**: TODO

### 8. Реализация **EmojiRenderer** (`src/render/emojiRenderer.js`)
**Описание**  
Методы:
- `showEmoji(gesture)` – отображает эмодзи в центре экрана на короткое время.  
- `showEmojiAt(gesture, landmarks)` – позиционирует эмодзи рядом с найденной рукой (используя координаты landmarks).

Все манипуляции с DOM находятся только в этом модуле.

**Definition of Done**  
- Эмодзи появляются с CSS‑переходами и исчезают после настраиваемого таймаута.  
- Нет утечек памяти (удаляются слушатели).  
- Визуальный тест подтверждает корректное позиционирование и плавное исчезание.

**Статус**: TODO

### 10. Реализация утилиты **Debounce** (`src/utils/debounce.js`)
**Описание**  
Чистая функция `debounce(fn, delay)`, возвращающая «заглушку», откладывающую вызов `fn` на `delay` мс.  
Должна предоставлять метод `cancel()`.

**Definition of Done**  
- Работает с любой сигнатурой функции.  
- Юнит‑тесты подтверждают тайминг и работу `cancel()`.

**Статус**: TODO

### 11. Связка всех компонентов в **app.js** (`src/app.js`)
**Описание**  
Последовательность:
1. Получить элемент `<video>` из DOM.  
2. `await initCamera(video)`.  
3. Инициализировать `HandTracker`.  
4. Создать экземпляр `ReactionState`.  
5. Запустить главный цикл:
   - `detect` → `detectGesture` → `reactionState.update` → `emojiRenderer.showEmoji…`.  
6. Глобальная обработка ошибок (отображать дружелюбные сообщения пользователю).

**Definition of Done**  
- Пайплайн работает от начала до конца без консольных ошибок.  
- Корректно обрабатывается отсутствие руки.  
- Работает в Chrome и Firefox.

**Статус**: TODO
