📏 4. Code_Quality.md

❗ 1. Принцип единственной ответственности

Каждый модуль делает только одну задачу.

❗ 2. Никакой логики в loop

Loop только управляет процессом.

❗ 3. Каждый жест — отдельный файл

❗ 4. Никакого DOM вне renderer

Работа с DOM только в слое рендера.

❗ 5. Константы в одном месте

❗ 6. Проверка на null

Всегда проверять входные данные.

❗ 7. Никаких магических чисел

Использовать именованные константы.

❗ 8. Чистые функции

Функции распознавания без side-effects.

❗ 9. Минимум глобального состояния

Не использовать window.

❗ 10. Понятные имена

Имена функций и переменных должны быть очевидны.

⚡ Требования к производительности
	•	≥ 20 FPS
	•	UI не должен лагать

🎯 UX требования
	•	Нет спама эмодзи
	•	Задержка < 100 мс

🛡 Надёжность
	•	Не падает без руки
	•	Обрабатывает ошибки корректно
